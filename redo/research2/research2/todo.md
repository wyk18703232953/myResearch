围绕 “竞赛题目→角色分配→多轮辩论→共识生成” 核心流程，展开以下研究：
2.2.1 辩论智能体角色框架设计
角色定义与职责：
算法专家：解析题目 Q 核心需求，输出 “算法思路 + 复杂度预估”（如 “动态规划，O (nlogn)，状态定义 dp [i] 为前 i 个元素最大和”），负责算法合理性；
实现专家：将算法思路转化为 C++/Python 代码，优化语言特性（如 Python 列表切片、C++ STL），输出代码与 “实现风险提示”（如 “第 12 行循环可能超时”）；
测试批判专家：设计边界测试用例（如输入为空、极值），批判算法逻辑（如 “未处理负权边，Dijkstra 算法无效”）与代码漏洞（如 “数组越界”），输出批判报告；
2.2.2 多轮辩论循环流程实现
辩论协议设计（四步循环）：
初始观点生成：每组的算法专家、实现专家、批判专家基于题目 Q 独立输出结果（算法思路 / 代码 / 批判报告），存入共享信息池（MySQL 数据库）；
组间观点交换与评审：不同组的算法专家交叉评审批判报告的合理性，实现专家交叉评审算法思路的可实现性，批判专家交叉评审代码与算法的一致性，输出评审意见；
反思与修正：每组角色基于其他组的评审意见修订结果 —— 如算法专家修正复杂度预估，实现专家修复代码漏洞，批判专家补充未覆盖的边界用例；
共识判断与聚合：由主控智能体（基于角色历史准确率加权投票，算法专家权重 0.4、实现专家 0.3、批判专家 0.3）判断是否达成共识（投票一致率≥80%）：是则输出最终结果，否则返回第 2 步继续辩论；
外部知识库调用：辩论中若角色需补充知识（如新型算法模板），自动触发 RAG 引擎检索竞赛知识库，确保观点专业性。
