## 实验总结

### 1. 实验概述

### 2. 动态运行时拟合方法

**核心思想**：通过实际运行代码并测量不同输入规模下的执行时间，使用数学模型拟合时间复杂度。

**技术实现**：
- **测试规模**：从 `start_n` 到 `max_n`，按 `step` 间隔递增（可配置）
- **重复测试**：每个输入规模重复测试3次，取中位数减少噪声
- **超时控制**：30秒超时机制
- **数据清洗**：
  - 移除冷启动效应（首次运行时间异常）
  - 基于Z-score移除异常值（阈值2.5）
  - 使用二次多项式拟合趋势线识别离群点

**拟合模型**：
- Constant: `f(n) = a + b`
- Linear: `f(n) = a·n + b`
- Logarithmic: `f(n) = a·log(n) + b`
- N Log N: `f(n) = a·n·log(n) + b`
- Quadratic: `f(n) = a·n² + c`
- Cubic: `f(n) = a·n³ + d`

**评估指标**：
- **R²（决定系数）**：衡量拟合优度
- **BIC（贝叶斯信息准则）**：模型选择，考虑模型复杂度
- **置信度分数**：综合R²、BIC权重、复杂度特征检测得分

**特殊优化**：
- **Constant检测**：基于变化幅度百分比（<5%判定为常数）
- **Logarithmic检测**：基于增长率递减趋势和对数拟合优度
- **N Log N检测**：基于增长率单调递增和显著变化幅度

**置信度计算公式**：
```
confidence_score = 0.5·R² + 0.3·BIC_weight + 0.2·feature_score - penalty
```

### 3. LLM静态分析方法

**核心思想**：利用大语言模型（GPT-5.1）分析代码结构，静态推断时间复杂度。

**技术实现**：
- **模型**：GPT-5.1（temperature=0.0确保确定性输出）
- **提示词设计**：
  - System prompt：明确要求只输出复杂度术语
  - User prompt：提供代码，要求分析时间复杂度
- **输出归一化**：将模型输出映射到标准术语（constant、linear、logn、nlogn、quadratic、cubic、exponential）

**复杂度等价映射**：
- constant ↔ {constant, o(1), 1}
- linear ↔ {linear, o(n), n}
- logn ↔ {logn, log n, o(log n), o(logn)}
- nlogn ↔ {nlogn, n log n, o(n log n), o(n logn)}
- quadratic ↔ {quadratic, o(n²), n²}
- cubic ↔ {cubic, o(n³), n³}

### 4. 融合方法

**核心思想**：结合LLM静态分析和动态运行时拟合的优势，通过智能融合策略提高检测准确率。

**融合策略**：

**4.1 质量评估**
- 评估拟合结果的可信度，包括：
  - R²值（拟合优度）
  - BIC差距（模型区分度）
  - 有效数据点数量
  - 置信度分数

**4.2 类型保护机制**
- **Constant类型**：严格保护
  - 置信度阈值：0.98
  - LLM差异阈值：0.40
  - BIC差距阈值：15
- **N Log N类型**：严格保护
  - 置信度阈值：0.98
  - LLM差异阈值：0.40
  - BIC差距阈值：12

**4.3 融合决策逻辑**

**情况1：LLM与拟合结果一致**
- 直接使用LLM结果

**情况2：LLM与拟合结果不一致**
- **Constant保护**：只有拟合结果极其可信且差异极大时才覆盖
- **N Log N保护**：只有拟合结果极其可信且差异极大时才覆盖
- **其他类型**：
  - 差异足够大（delta_llm ≥ 0.20）且拟合可信时使用拟合结果
  - 置信度极高（≥0.95）时使用拟合结果
  - 置信度较高（≥0.85）且拟合可信时使用拟合结果

**4.4 LLM复查机制**
- 只有当置信度低于0.80时才进行复查
- 复查结果决定最终输出

**4.5 通用参数**
- R²阈值：0.95
- BIC差距阈值：10
- 置信度阈值：0.90
- 有效数据点阈值：10
